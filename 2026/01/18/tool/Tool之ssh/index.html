SSH 簡介

定義： SSH (Secure Shell) 是一種加密的網路傳輸協定。
用途： 用於在不安全的網路（網際網路）上，安全地操作遠端電腦。
前身： 取代了早期的 Telnet。Telnet 是明文傳輸（任何人用 Wireshark 都能看到你的密碼），而 SSH 是全加密的。
預設 Port： TCP 22。
運作模式： Client-Server 架構。你的電腦是 Client (ssh 指令)，伺服器端運行 Daemon (sshd)。

常用指令

基礎連線 (Basic Connection)

最基本的登入方式。

指定端口 (Specify Port)

很多伺服器為了安全，不會開在預設的 22 Port。

使用金鑰登入 (Key-Based Auth)

這是 AWS/GCP 等雲端伺服器的標準登入方式。

執行單次指令 (One-Shot Command)

你不需要登入 Shell 也能執行指令。這適合自動化腳本。

金鑰管理 (Key Management)

密碼容易被暴力破解，金鑰則幾乎不可能。

產生金鑰對 (Generate Keys)
在你的電腦上製造一把鎖（公鑰）和一把鑰匙（私鑰）。

部署公鑰 (Deploy Public Key)
將你的「鎖」安裝到伺服器上，以後你拿著「鑰匙」就能直接進去，不用打密碼。

隧道與跳板 (Tunneling & Proxy)

這是 SSH 最強大的功能。它可以把你的流量「藏」在 SSH 連線裡面，繞過防火牆。

SSH 隧道 (Tunneling / Port Forwarding)

情境： 假設你攻陷了一台網頁伺服器 (Server A)。 這台 Server A 內部還有一個 資料庫 (Database)，跑在 Port 3306。 但是！管理員設了防火牆，只允許 Port 22 (SSH) 通行，其他路都被封死了。你無法直接連線到 3306 偷資料。
解決方案： 既然 Port 22 是唯一暢通的路，那我們就把「資料庫的連線」塞進 SSH 這個地道裡偷渡出來。

-L (Local): 在我本地電腦開一個入口。
9999: 在本地開9999 port
localhost:3306: 出口要接到伺服器那邊的 3306。
結果： 現在，你只要連線到你自己電腦的 localhost:9999，SSH 就會幫你把訊號透過地道傳送到伺服器的 3306。 防火牆只看到你在用 SSH，完全不知道裡面包裹著資料庫的傳輸。

動態隧道 (Dynamic Forwarding / SOCKS Proxy)

情境： 你在星巴克用 Wi-Fi，但你不想讓星巴克知道你在看什麼網站。或者你想去瀏覽一個只有美國 IP 才能看的網站。
解決方案： 你把遠端的 SSH 伺服器當成你的「代理人」。

結果： 我所有要上網的流量，都先丟進這個 1080 地道。這些流量會從遠端伺服器那邊冒出來，再連去 Google 或 Netflix。

SSH 跳板 (Jump Host)

情境： 大多數企業為了安全，重要的主機（比如存客戶個資的資料庫 Server B）是完全沒有對外網路的。你無法直接 SSH 連進去。 但是，有一台比較不重要的機器（Server A，我們稱為 Bastion Host 或跳板機）是可以從外部連線的，而且 Server A 可以連到 Server B。
路徑： 你 -> Server A (跳板) -> Server B (目標)。
解決方案： 以前我們很笨，要先 SSH 進 A，然後在 A 裡面再打一次 SSH 連去 B。 現在 SSH 有一個超強參數 -J (Jump)。

結果： 直接登入了 Server B，中間的跳轉過程 SSH 自動幫你處理好了。
